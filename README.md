# EEGFlow

マインドアップロード実現のための中核となるコアサイト。

## 目的

このサイトは、マインドアップロードの実現に向けた技術・研究・コミュニティの中心的ハブとなることを目指しています。

ここでいうマインドアップロードとは、**意識や記憶の交換・複製を可能とする技術**を指します。

## 今後の展望

本プロジェクトは、最終的にサイトの更新や運用プロセスを完全に自動化することを目指しています。現在は手動で行っているタスクも、順次自動化ツールやCI/CDパイプラインへ移行する予定です。

## データ標準と倫理・法規制 (Data Standards, Ethics, and Regulation)

マインドアップロード研究は、その倫理的・法的課題の克服なしには成立しません。本プロジェクトは、技術開発と並行して、堅牢なデータガバナンスと倫理的フレームワークの構築を最重要課題と位置づけます。

- **データ標準と再現性**:
  - **BIDS (Brain Imaging Data Structure)**: すべてのデータとメタデータはBIDS標準に準拠させ、構造的な一貫性と共有可能性を確保します。
  - **再現可能な解析パイプライン**: BIDS-App準拠の解析パイプラインを導入し、解析プロセスの透明性と再現性を保証します。

- **法的・倫理的枠組みの遵守**:
  - **MIND Actの動向**: 米国で提案された**MIND Act (S.2925)** を注視していますが、これが現時点では**提案段階（Introduced）の法案**であり、確立された規制ではないことを明確に認識しています。我々はその進展を監視し、将来の規制の方向性を見据えます。
  - **国際的整合性**: 日本の**個人情報保護法**や、ユネスコが提唱する**ニューロライツ（神経権利）**といった国際的な議論と調和した運用を目指します。

- **具体的なデータガバナンス戦略**:
  - **高度なプライバシー保護技術**: 神経データ特有の再識別化リスクに対応するため、以下の技術を導入します。
    - **匿名化**: MRIデータから顔形状を削除する[PyDeface](https://github.com/poldracklab/pydeface)などを活用します。
    - **差分プライバシー (Differential Privacy)**: データ解析時に個人のプライバシーを数学的に保証する差分プライバシーを適用します。具体的には、プライバシー損失の許容量を示す**ε（イプシロン）値を定義・公表**し、解析の有用性とプライバシー保護のトレードオフを透明化します。
  - **動的な同意管理**: データ提供者が、自身のデータの利用目的や共有範囲をきめ細かく、かつ動的に制御できる同意管理システムを構築します。

## システム構成 (System Architecture)

本プロジェクトでは、AIエージェントを活用した半自動的なコンテンツ更新ワークフローを採用しています。

```mermaid
graph LR
    A[ユーザー] -->|Issue作成| B(Manus AI)
    B -->|Issue登録| C[GitHub Issues]
    C -->|Issue取得| D(Antigravity)
    D -->|コード修正・コミット| E[GitHub Repository]
    E -->|自動デプロイ| F[GitHub Pages]
    F -->|閲覧| A
```

### ワークフロー

1.  **Issue作成 (Manus)**: ユーザーがManus AIに対して改善提案や新機能のリクエストを伝えると、ManusがGitHub Issueを自動作成します。
2.  **Issue処理 (Antigravity)**: Antigravity（本エージェント）がオープンなIssueを取得し、コードベースを分析・修正し、コミット＆プッシュを行います。コミットメッセージに `Fixes #N` を含めることで、Issueは自動的にクローズされます。
3.  **デプロイ (GitHub Pages)**: `main` ブランチへのプッシュをトリガーに、GitHub Pagesが自動的にサイトを更新します。
